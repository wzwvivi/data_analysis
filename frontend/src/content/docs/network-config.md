# TSN 网络配置管理

**TSN 网络配置管理**（曾用名「TSN 网络配置」「TSN 网络协议管理」；开发侧路由仍为 `/network-config`）负责网络侧配置资产的版本化管理，支持草稿编辑、审批流和版本查看。

## 价值

- 将 ICD/端口/字段等网络侧解析口径固化为可发布版本，随发布升级而演进。
- 草稿、审批、发布三态与设备协议管理平行，便于双轨治理。

## step-1 浏览与维护

1. 在模块首页查看协议与版本列表。
2. 需要变更时，从源版本创建草稿，或从 ICD Excel 导入新草稿。

## step-2 提交与发布

1. 完成草稿后提交审批，按团队流程会签或终审。
2. 审批通过后，由有权限角色发布为可用版本（或 PendingCode 待激活，依部署说明）。

## 输入与输出

- 输入：本模块中的网络侧协议草稿、审批信息。
- 输出：可被选中的协议/配置版本，供「上传解析」、专项分析、TSN 异常检查等流程绑定。

## 常见问题

### 新端口未解析到

- 确认已选用包含该端口的 TSN 网络侧协议/配置版本，且对应设备协议已激活。

### 与设备协议关系

- TSN 网络侧配置管端口/字段/拓扑；ARINC 429 等设备侧细粒度在「设备协议管理」中维护，二者需版本配套使用。
