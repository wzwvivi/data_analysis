# 快速开始

本指南帮助新用户在约 10 分钟内完整走通平台主链路：**登录 → 上传解析 → 任务追踪 → 结果分析**。建议首次使用时准备一份 100MB~500MB 的样例数据文件，便于快速熟悉流程。

## 前置准备

1. 已获得平台访问地址与账号（由管理员开通）。
2. 账号已分配所需的页面权限（至少包含 `上传`、`任务中心`，以及目标分析模块）。
3. 准备好待解析的原始数据文件；建议首次使用小体积样例文件。

## step-1 登录与权限确认

1. 打开平台地址，进入登录页。
2. 输入账号和密码完成登录。
3. 进入平台后，检查左侧菜单是否包含目标模块；若缺少入口，请联系管理员确认权限配置。

## step-2 提交解析任务

1. 在左侧菜单进入 `网络数据分析 → 上传解析`。
2. 选择原始数据文件，填写任务名称与必要参数。
3. 选择对应的 **协议版本** 与 **端口范围**（端口下拉受账号可见范围限制）。
4. 点击提交，平台会生成任务编号并进入排队或执行状态。

## step-3 跟踪任务状态

1. 进入 `任务中心`，按时间或状态筛选定位到刚提交的任务。
2. 状态含义：
   - `排队中`：等待后台解析资源。
   - `运行中`：后台正在解析。
   - `成功`：已生成可用结果。
   - `失败`：任务异常中止，点击查看错误详情。
3. 失败任务建议先核对文件、协议版本与端口选择，再决定是否调整参数重新提交。

## step-4 进入结果分析

根据业务目标选择进入对应模块：

- 查看总体联查：`试验工作台`。
- 查看事件链路：`飞管事件分析` 或 `飞控事件分析`。
- 查看自动飞行指标：`自动飞行性能分析`。
- 比对任务差异：`TSN 异常检查`。

也可直接从任务详情页点击快捷入口跳转。

## step-5 沉淀结论

1. 标注关键发现、异常窗口与对应证据。
2. 将常用筛选条件保存到浏览器书签或团队文档。
3. 对需要跨任务跟踪的问题，记录任务编号与分析路径，便于后续回溯。

## 常用快捷入口

- 顶部 `?` 按钮：一键打开帮助中心。
- 菜单 `帮助与支持 → 帮助中心`：浏览模块化文档。
- 帮助中心右上角 `打印`：将当前文档打印或导出为 PDF。

## 常见问题

### 上传后任务一直排队

平台对解析并发有保护阈值，高峰期会排队。通常稍等数分钟即可开始执行；若长时间无变化，请联系管理员核查。

### 看不到某个模块入口

当前账号未开通该页面权限。请联系管理员在 `用户管理` 中调整权限配置。

### 任务失败原因如何确认

进入任务详情页，查看错误信息与任务参数。多数失败源自文件格式不匹配、协议版本选择错误或端口超出可见范围。
