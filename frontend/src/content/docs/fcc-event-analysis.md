# 飞控事件分析

飞控事件分析用于定位飞控相关事件与状态变化，帮助快速定位控制链路中的风险点。

## 典型场景

- 飞控异常触发后的根因初筛。
- 关键控制状态切换时序验证。
- 对比不同任务中的飞控事件模式。

## step-1 打开任务分析页

1. 从模块首页创建分析任务或从历史任务进入详情。
2. 确认数据范围和任务标识。

## step-2 查看事件结果

1. 按类型、等级和时间过滤事件。
2. 查看事件详情中的关键字段与上下文。
3. 必要时跳转工作台做补充验证。

## step-3 输出结论

1. 标注关键事件证据。
2. 记录建议动作（回归、复测、规则修订）。

## 输入与输出

- 输入：解析结果、飞控事件规则。
- 输出：事件任务详情、筛选结果和证据条目。

## 常见问题

### 分析任务创建失败

- 检查源任务是否可用。
- 检查后台资源是否繁忙或参数缺失。
