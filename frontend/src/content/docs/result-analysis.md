# 结果分析

解析任务成功后，可从任务详情进入 `结果分析` 页面。该页面用于将解析后的结构化数据按端口、设备和字段进行可视化查看，是“上传解析 → 任务中心”之后最常用的分析入口之一。

## 模块价值

- 将解析结果从原始表格进一步转化为可观察的图表趋势。
- 支持按端口、设备、解析器切换视角，快速定位目标数据。
- 支持多字段叠加和跨设备对比，便于判断变化是否异常。
- 与异常分析联动，可从趋势观察进一步进入跳变、卡死等问题排查。

## 适合查看什么

- 某个字段随时间的变化趋势，例如高度、航向、速度、电压、电流等。
- 同一端口下多个字段之间的联动关系。
- 不同设备或不同端口在同一时间轴上的表现差异。
- 某个异常时间点前后的数据波动情况。

## 页面结构说明

结果分析页通常包含以下三个部分：

### 数据表格

- 查看解析后的明细记录。
- 支持切换当前端口。
- 可按 `Label / CAN ID` 快速筛选列和行。
- 支持导出 `CSV`、`Parquet`，也可通过列管理控制显示字段。

### 数据分析

这是图表展示的核心区域，主要用于画图查看字段趋势。

- **同端口多字段**：在同一个端口下选择多个数值字段，观察它们在同一时间轴上的变化。
- **跨设备对比**：选择多个设备或端口，对比相同字段或不同字段的趋势差异。
- 支持预加载字段、快速选择字段，并根据分析需要切换不同的对比方式。

当你想回答“这个值什么时候开始变化”“两个参数是否同步波动”“不同设备表现是否一致”这类问题时，优先使用这里的图表分析。

### 异常分析

在图表趋势之外，页面还提供异常分析能力，用于辅助识别：

- **跳变**：字段值相对预测值出现明显偏离。
- **卡死**：字段在一段时间内持续不变化。

如果图表中已经看到可疑波动，可以继续在异常分析页中缩小排查范围。

## 推荐使用步骤

1. 在 `任务中心` 打开一个解析成功的任务。
2. 先在 `数据表格` 中确认目标端口、字段是否存在。
3. 切换到 `数据分析`，选择需要观察的数值字段。
4. 先看单字段趋势，再逐步增加对比字段，避免一次选太多影响判断。
5. 发现异常波动后，再进入 `异常分析` 查看跳变或卡死结果。

## 使用建议

- 首次分析时，建议优先选择 1~3 个关键字段，便于快速建立判断。
- 做跨设备对比时，尽量选择含义一致、量纲一致的字段。
- 如果图表信息过多，可先回到数据表格缩小端口或字段范围。
- 对重要结论，建议同时记录任务编号、端口信息和异常时间点，便于后续复盘。

## 常见问题

### 为什么有些字段不能画图

图表分析主要面向可连续观察的数值字段。文本类、说明类或非数值字段通常不参与趋势绘制。

### 图表里看不到想要的字段

- 先确认当前切换到的是正确端口。
- 再确认该字段是否属于当前解析结果中的可分析数值字段。
- 若仍未显示，建议回到数据表格确认字段是否已被成功解析。

### 什么时候看图表，什么时候看异常分析

图表更适合做趋势判断和人工观察；异常分析更适合做自动筛查和快速定位。实际工作中通常先看图，再用异常分析收敛问题范围。
